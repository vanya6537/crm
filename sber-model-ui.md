В интерфейсе **SberCRM** (в частности, в модальном окне **«Новое поле»** в Менеджере объектов → Менеджер полей) drag-and-drop и визуальная структура реализованы следующим образом (описание основано на типичном поведении из скриншотов и видео-демонстраций системы на 2025–2026 годы).

### Общий визуальный стиль модального окна «Новое поле»
- Окно появляется поверх основного интерфейса (overlay / modal dialog) с сероватым затемнением фона.
- Заголовок окна: **«Новое поле»** (белый фон, тёмно-серый текст, шрифт sans-serif ~16–18 px).
- Кнопка закрытия: крестик (×) в правом верхнем углу, серый, при наведении становится чёрным/синим.
- Основной фон окна — белый (#FFFFFF).
- Разделители и заголовки групп — серые линии и чуть более тёмные заголовки (тёмно-серый #333–#444).
- Кнопки внизу: **Отмена** (серый контур) и **Добавить** (зелёная заливка, белый текст, hover → более яркий зелёный).

### Главная область — карточки типов полей (самый важный drag & drop элемент)

Поля разделены на 3–4 вертикальные колонки (группы), каждая группа имеет заголовок:

1. **Текстовые поля и справочники**
2. **Числа, даты и время**
3. **Связь объектов** (самая важная и визуально выделенная)
4. **Специальные поля** (иногда отдельно или внизу)

Каждая карточка типа поля — это **прямоугольная карточка** размером примерно 220–260 × 120–140 px:

- Белый фон с тонкой серой рамкой (#E0E0E0).
- При наведении (hover):
  - лёгкая тень (box-shadow),
  - рамка становится синей (#0066FF или корпоративный синий Сбера),
  - курсор превращается в руку (pointer).
- Внутри карточки:
  - сверху — иконка (маленькая svg-иконка 24×24 px, серо-синяя)
  - посередине — пример визуализации поля (имитация ввода)
  - снизу — название типа (например «Короткий текст», «Дата и время», «Связь»)
- Примеры имитаций внутри карточек:
  - Короткий текст → прямоугольник с серым плейсхолдером «Короткий текст здесь…»
  - Длинный текст → высокий прямоугольник с полосами имитирующими несколько строк
  - Дата → «ДД.ММ.ГГГГ»
  - Продолжительность → «0 дн. 0 ч. 0 мин.»
  - Связь → две квадратные иконки человечков соединённые стрелками (↔ или →)
  - Мастер-связь → то же, но с пометкой «i» и жирной стрелкой

**Drag & drop в этой области практически отсутствует** — карточки типов **нельзя перетаскивать**.  
Они статичны, выбор происходит простым кликом по карточке → она подсвечивается синим и справа появляются поля настроек (Имя, Описание, Связанная таблица и т.д.).

### Drag & drop появляется в других местах системы (не в окне «Новое поле»)

1. **Вкладка «Макеты»** (Менеджер объектов → выбранный объект → вкладка Макеты)
   - Здесь настоящий drag-and-drop.
   - Слева — список доступных полей (в виде вертикального списка карточек или строк).
   - Справа — область формы/макета (разделённая на секции / табы / шаги).
   - Пользователь перетаскивает поля из левого блока в правый → они становятся частью формы.
   - Можно менять порядок полей внутри формы перетаскиванием (вертикальный drag).
   - При перетаскивании появляется полупрозрачная копия карточки, след движется за курсором.
   - Зоны дропа подсвечиваются синим/зелёным контуром.

2. **В списке полей объекта** (Менеджер полей → Активные поля)
   - Поля отображаются в таблице или карточками.
   - Есть иконка «≡» или шесть точек (grip dots) слева от каждого поля.
   - Перетаскивание этих иконок позволяет менять порядок полей в списке (вертикальный drag-and-drop).
   - После изменения порядка нужно нажать «Сохранить» или происходит автосохранение.

3. **В связях множественного выбора** (когда уже создано поле типа «Связь множественного выбора»)
   - В форме карточки объекта появляется блок с кнопкой «Добавить» → открывается модалка выбора/создания связанных объектов.
   - Выбранные связанные записи можно перетаскивать для изменения порядка (drag handles в виде ≡).

### Всплывающие подсказки (tooltips) и иконки «i»
- Почти каждая карточка типа и каждое поле настройки имеет маленькую иконку **«i»** в кружке (серый).
- При наведении появляется tooltip (белый прямоугольник с тенью, текст 12–14 px):
  - Для «Мастер-связь» → «Проставить Мастер-связь на объект. При удалении основного объекта также будет удалён связанный объект».
  - Для «Связь множественного выбора» → «Добавить на форму или список поле с возможностью… создавать до 50 связанных объектов…».
- Подсказки появляются с небольшой задержкой (~300–500 мс), исчезают при уводе курсора.

### Цветовая схема (корпоративный стиль Сбера)
- Основной акцент — синий (#0055FF / #0066CC)
- Успешные действия / кнопки — зелёный (#00CC66)
- Ошибки / обязательные поля — красный (#FF3333, красная полоса слева от поля)
- Неактивные элементы — серый (#999)

Если коротко:  
В самом окне **«Новое поле»** drag-and-drop **нет** — выбор типа поля кликом по карточке.  
Настоящий drag-and-drop присутствует в **Макетах форм** и в **порядке полей** в списке → там перетаскивание визуально приятное, с плавной анимацией, подсветкой зон дропа и grip-дотами.

Если нужно — могу описать конкретный сценарий (например, как именно выглядит перетаскивание в макетах) более детально.
