Вот подробное описание функционала интерфейса **создания нового поля** в объекте (на примере объекта «Контакты» / Individual) в SberCRM — в формате, подходящем для технического задания / описания функциональных требований.

### 1. Общее расположение и запуск функции
- Раздел: **Настройки → Менеджер объектов**
- Объект: выбирается из списка (в примере — «Контакты» / Individual)
- Вкладка: **Менеджер полей**
- Кнопка: **+ Новое поле** (в правом верхнем углу или рядом с табами)

### 2. Основные элементы модального окна «Новое поле»

| Элемент                        | Описание                                                                                 | Обязательность | Примечание / ограничения |
|-------------------------------|------------------------------------------------------------------------------------------|------------------|----------------------------------|
| Имя *                          | Отображаемое название поля в интерфейсе                                                  | Обязательно      | —                                |
| Описание                       | Подсказка / пояснение для пользователей (отображается при наведении на i-иконку)        | Нет              | —                                |
| Системный справочник           | Выбор из готовых системных справочников (Да/Нет, Страна, Валюта и т.д.)                 | Нет              | Автоматически заполняет значения |
| Пользовательский справочник    | Возможность создать / выбрать свой справочник прямо из формы                            | Нет              | Кнопка «+ Добавить пользовательский справочник» |
| Связанная таблица *            | Обязательное поле при выборе типов связи                                                | Условно обяз.    | Появляется только для типов «Связь» |

### 3. Группы типов полей (основные категории)

#### 3.1. Текстовые поля и справочники
- Короткий текст
- Длинный текст (многострочное поле)
- Текст (обычное однострочное)
- Большой текст (расширенное многострочное)
- Справочник (выбор одного значения из списка)
- Выберите вариант (аналогично справочнику, но с другим визуальным стилем)
- Да/Нет (булевый тип, часто системный)

#### 3.2. Числа, даты и время
- Число (целое или с плавающей точкой)
- Дата (ДД.ММ.ГГГГ)
- Дата и время (ДД.ММ.ГГГГ чч:мм)
- Продолжительность (0 дн. 0 ч. 0 мин.)

#### 3.3. Связь объектов (самая важная и сложная группа)
| Тип связи                              | Описание                                                                                     | Кол-во связанных записей | Поведение при удалении связанного объекта | Когда использовать |
|----------------------------------------|----------------------------------------------------------------------------------------------|---------------------------|--------------------------------------------|---------------------|
| **Связь**                              | Обычная связь 1 объект ↔ 1 объект или 1 объект → много объектов                            | 1 или много               | Зависит от настройки мастера               | Почти все случаи связи |
| **Мастер-связь**                       | Главная (master) связь. При удалении основного объекта удаляются и все связанные записи    | 1 или много               | Каскадное удаление                         | Телефоны, email-ы, адреса, представители |
| **Связь множественного выбора**        | Позволяет выбрать/создать сразу несколько связанных объектов прямо в форме создания        | Много (до 50 за раз)      | Не каскадное (по умолчанию)                | Несколько телефонов, несколько задач, несколько представителей |

**Важные механики связей:**
- При выборе типа «Связь» или «Мастер-связь» обязательно указывается **Связанная таблица** (из списка всех объектов системы).
- Мастер-связь помечается специальной иконкой и всплывающей подсказкой:  
  «Проставить Мастер-связь на объект. При удалении основного объекта также будет удалён связанный объект».
- Связь множественного выбора позволяет создавать до 50 связанных объектов сразу при создании основной записи.

#### 3.4. Специальные поля (дополнительные типы)
- Телефон
- E-mail
- Ссылка (URL)
- Нумератор (автонумерация)
- Один из списка
- Чек-лист
- Файл (загрузка)
- Чек-лист задач (с подзадачами)

### 4. Дополнительные настройки (блок «Дополнительно»)

- Разрешить выбор нескольких значений (флажок) — для справочников и связей
- Обязательное поле (системно проставляется красной чертой и надписью)
- Видимость / доступность для ролей (настраивается отдельно во вкладке Макеты)
- Формулы / вычисляемые значения (настраиваются позже во вкладке формул)
- Валидация (маска, регулярное выражение и т.д.) — вкладка «Настройка валидаций»

### 5. Кнопки управления в окне создания поля
- **Отмена** — закрытие без сохранения
- **Добавить** — создание поля и закрытие окна

### 6. После создания поля (где появляется)
- Вкладка **Менеджер полей** → список активных полей объекта
- Возможность перетаскивания для изменения порядка
- Переключение **Активные / Архивные**
- Доступны дальнейшие настройки:
  - Макеты (разные наборы полей для разных ролей)
  - Зависимости полей
  - Настройка валидаций
  - Настройка стадийной модели
  - Менеджер меток
  - Группы уникальных полей

### Рекомендуемая структура для ТЗ (кратко)

**Функциональное требование № [X]**  
**Название:** Создание пользовательского поля в объекте  
**Модуль:** Менеджер объектов → [Название объекта] → Менеджер полей  
**Актер:** Администратор системы  
**Предусловие:** Пользователь имеет роль с правами на изменение метаданных  
**Основной сценарий:**
1. Открыть список полей объекта
2. Нажать «+ Новое поле»
3. Выбрать категорию и тип поля
4. Заполнить обязательные атрибуты (Имя, Связанная таблица — для связей)
5. Указать дополнительные параметры (мастер-связь, множественный выбор, справочник и т.д.)
6. Нажать «Добавить»

**Альтернативные сценарии:**
- Выбор мастера-связи → автоматическое каскадное удаление
- Выбор множественной связи → лимит 50 создаваемых записей за раз
- Выбор системного справочника → значения подтягиваются автоматически

Если нужно — могу дополнить конкретными примерами полей, которые чаще всего создают в «Контактах», «Организациях», «Сделках» и т.д.
