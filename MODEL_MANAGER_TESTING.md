# Инструкция по тестированию Model Manager

## 🚀 Быстрый старт

### 1. Убедитесь, что сервер запущен
```bash
# Сервер должен запуститься на http://127.0.0.1:3000
# Проверить статус: ps aux | grep "serve"
```

### 2. Откройте в браузере
```
http://127.0.0.1:3000/model-manager
```

## 📋 Чек-лист функционала

### ✅ Тест 1: Создание поля
1. Откройте Model Manager
2. Выберите "Агенты" в левой панели
3. Нажмите "+ Новое поле"
4. Заполните:
   - Name: `test_field` 
   - Label: `Test Field`
   - Field Type: `text`
5. Нажмите "Добавить"
6. **Проверка БД**:
   ```bash
   sqlite3 storage/database.sqlite "PRAGMA table_info(agents);" | grep test_field
   ```
   Должен вывести строку с `test_field|`.

### ✅ Тест 2: Перетаскивание полей
1. В списке полей наведите на левый иконка ≡ (grip-dots)
2. Нажмите и удерживайте на иконке
3. Перетащите поле на новую позицию (должна быть синяя подсветка)
4. Отпустите
5. **Проверка**: Поле переместилось в новую позицию, должно быть уведомление "Порядок сохранён"

### ✅ Тест 3: Редактирование поля
1. Нажмите кнопку "Изменить" (синяя) на любом поле
2. Измените параметры (напр. Description)
3. Нажмите "Обновить"
4. **Проверка**: Поле обновилось в списке

### ✅ Тест 4: Удаление поля
1. Нажмите кнопку "Удалить" (красная) на поле `test_field`
2. Подтвердите удаление
3. **Проверка БД**:
   ```bash
   sqlite3 storage/database.sqlite "PRAGMA table_info(agents);" | grep test_field
   ```
   Не должно быть результатов (столбец удалён).

### ✅ Тест 5: Архивирование поля
1. Нажмите кнопку архить (оранжевая) на любом поле
2. Поле должно исчезнуть из активного списка
3. Переключитесь на "Интернет" (tabs) и нажмите "Архив"
4. Поле должно появиться в архиве с кнопкой "Восстановить"
5. Нажмите "Восстановить"
6. Поле вернётся в активный список

### ✅ Тест 6: Переключение между сущностями
1. Нажмите на "Недвижимость" слева
2. Список полей обновится (будут другие поля для properties)
3. Попробуйте добавить поле для недвижимости
4. **Проверка БД**:
   ```bash
   sqlite3 storage/database.sqlite "PRAGMA table_info(properties);"
   ```
   Новое поле должно быть в таблице properties.

## 📊 Примеры curl запросов

### Создать поле программно
```bash
curl -X POST "http://127.0.0.1:3000/api/v1/model-fields/agent" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "middle_name",
    "label": "Middle Name",
    "field_type": "text",
    "required": false
  }'
```

### Изменить порядок полей
```bash
curl -X POST "http://127.0.0.1:3000/api/v1/model-fields/agent/reorder" \
  -H "Content-Type: application/json" \
  -d '{
    "fields": [
      {"id": "uuid-1", "sort_order": 0},
      {"id": "uuid-2", "sort_order": 1}
    ]
  }'
```

## 🛠 Отладка проблем

### Проблема: API возвращает "Unauthenticated"
**Решение**: Сейчас API требует auth token. Используйте браузер для тестирования UI (работает через session).

### Проблема: Фронтенд не обновляется после создания поля
**Решение**: 
1. Обновите страницу (Cmd+R)
2. Проверьте консоль браузера (F12 → Console)
3. Проверьте логи сервера: `tail -f storage/logs/laravel.log`

### Проблема: "Field name already exists"
**Решение**: Используйте уникальное имя для каждого тестового поля.

### Проблема: Drag-and-drop не работает
**Решение**: 
1. Убедитесь, что беретесь за иконку ≡ (grip-dots)
2. Медленнее перетаскивайте
3. Проверьте браузер (нужна поддержка HTML5 Drag API)
